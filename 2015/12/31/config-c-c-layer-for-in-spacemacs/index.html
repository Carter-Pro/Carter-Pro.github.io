Configure c/c++ layer for pure c in Spacemacs

发表于 2015-12-31 | 分类于 Emacs | 阅读次数:

前言

最近因为实验室项目和自己学习混合使用了很多语言，加上非常想尝试用Org-mode作为自己的GTD管理工具，因此选择了Emacs作为编辑器。

初次使用Emacs比较好的方法是使用大牛们的配置。最终我选择了Spacemacs，因为它有广泛的用户群体、合理的组织结构、丰富的用户文档，非常适合Emacs初学者，最重要的是它颜值高

内容

根据Spacemacs上c-c++ layer的说明可以很轻松的完成配置，但是需要注意的是Spacemacs很多默认是支持c++。

目前我使用的主力语言还是c，因此本文主要描述对c语言支持的配置。

除了按照c/c++ layer配置完之外，还需要注意三点：

  1. 想要使用自动补全和动态语法检查还需要参考ycmd和syntax-checking两个layer的文档，他们都可以通过安装本地ycmd-server和配置ycmd来实现
  2. 由于新的Mac OS X（记不清是10.10还是10.11之后了）取消了/usr/include和/usr/local/include两个文件夹，还需要为company-c-header配置环境变量使之支持头文件补全
  3. Ycmd和company-c-header配合不完善，导致company-c-header对头文件的补全会被ycmd截断，需要更改一下配置文件。
