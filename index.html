整数的取模和取余

Posted on 2016-02-27 | In 思考

今天碰到一个涉及到负数取模的问题不知道怎么算了。在研究的过程中发现自己以往一直把取模(modulus)和取余(remainder)当做一个概念用，但是这两个概念是不是一样的呢？

参考

  1. 负数取模怎么算
  2. 实数范围内的求模（求余）运算：负数求余究竟怎么求
  3. What’s the difference between “mod” and “remainder”?

数学上取模与取余

取模和取余的步骤

对于整数a和b来说，取模运算和取余运算步骤是一样的。

  1. 求整数商c

    $$
    c = a / b
    $$

    ​

  2. 计算模/余数r

    $$
    r = a - c * b
    $$

取模和取余区别

取模和取余的区别在于第一步计算整数商c时取整的方法不一样。

整数除法取整的3种方法

  1. 向上取整，向+∞方向取最接近精确值的整数，也就是取比实际结果稍大的最小整数，也叫 Ceiling 取整。这种取整方式下，17 / 10 == 2，5 / 2 == 3, -9 / 4 == -2。
  2. 向下取整，向-∞方向取最接近精确值的整数，也就是取比实际结果稍小的最大整数，也叫 Floor 取整。这种取整方式下，17 / 10 == 1，5 / 2 == 2, -9 / 4 == -3。
  3. 向零取整，向0方向取最接近精确值的整数，换言之就是舍去小数部分，因此又称截断取整（Truncate）。这种取整方式下，17 / 10 == 1，5 / 2 == 2, -9 / 4 == -2。

区别

其中，取余使用了向零取整，取模使用了向下取整。

因此，在被除数与除数符号一致的情况下，取模与取余结果是相同的。

例子

例如：计算-7 Mod 4

那么：a = -7；b= 4；

  1. 求整数商c。如进行求模运算c = -2（向负无穷方向舍入），求余c = -1（向0方向舍入）；
  2. 计算模和余数的公式相同，但因c的值不同，求模时r = 1，求余时r = -3。

计算机的取模和取余

在流行编程语言中（除了matlab），%就是取余（至少在C、C++、Java和C#是这样的），教科书将这两个概念同化了。

看一下C和Python计算上述例子。

语言 计算结果
C -3
Python 1

因为C语言在整除时使用了向零取整，Python使用了向下取整。

逻辑地址、虚拟地址、线性地址和物理地址

Posted on 2016-02-07 | In 操作系统

在学习xv6的过程中，我遇到了一个很容易混淆的问题，就是逻辑地址、虚拟地址、线性地址和物理地址这四种地址有什么区别、分别作用在哪里。

首先，我们看一张图片。

该图片来自《Linux内核完全剖析——基于0.12内核》的第4.2节保护模式内存管理。

看完上图我有3个问题：

  1. 这4种地址的定义是什么？
  2. 虚拟地址和逻辑地址有什么区别？
  3. 线性地址和物理地址有什么区别？

4种地址的定义

首先我们先看看四种地址的定义。

物理地址

物理地址（physical address），也叫实地址（real address）、二进制地址（binary address），它是在地址总线上，以电子形式存在的，使得数据总线可以访问主存的某个特定存储单元的内存地址。

也就是说，物理地址是地址总线上的寻址物理内存的地址信号。但是往往会将其看做物理内存某特定存储单元的内存地址。

线性地址

线性地址（Linear Address）是逻辑地址到物理地址变换之间的中间层。

线性地址和物理地址有什么区别

如果开启了分页机制，线性地址需要再经过变换产生一个物理地址；如果没有启用分页机制，线性地址就是物理地址。其中，如果开启分页机制，线性地址会拆成页号和偏移地址再转换成物理地址。

逻辑地址

在有地址变换功能的计算机中,访问指令给出的地址 (操作数) 叫逻辑地址,也叫相对地址。要经过寻址方式的计算或变换才得到内存储器中的物理地址。

上述定义有点难以理解，但是在计算机体系结构中，逻辑地址是指应用程序角度看到的内存单元。逻辑地址由段号(段名)和段内地址所组成。

举个例子。在分段存储管理方式中,作业的地址空间被划分为若干个段,每个段定义了一组逻辑信息。例如,有主程序段 MAIN、子程序段 X、数据段 D 及栈段 S 等,如下图所示。每个段都有自己的名字。为了实现简单起见,通常可用一个段号来代替段名,每个段都从 0开始编址,并采用一段连续的地址空间。段的长度由相应的逻辑信息组的长度决定,因而各段长度不等。整个作业的地址空间由于是分成多个段,因而是二维的。

虚拟地址

关于虚拟地址的定义基本没有，虚拟地址的概念是相对于物理地址来说的。在百度百科虚拟地址中是这么描述的：

虚拟地址 (virtual address): CPU启动保护模式后，程序运行在虚拟地址空间中。

虚拟地址和逻辑地址有什么区别

所为我们可以认为虚拟地址是一个抽象的概念，在不同书籍中其被使用的地方也不同。其中，在《深入理解linux内核》中线性地址被称为虚拟地址，而《Linux内核完全剖析》中逻辑地址被称为虚拟地址。

四种地址关系的分析

从图1我们可以看到80x86在从逻辑地址到物理地址变换过程中使用了分段和分页两种机制。第一阶段使用分段机制把程序的逻辑地址变换成处理器可寻址内存空间（成为线性地址空间）中的地址。第二阶段使用分页机制；把线性地址转换为物理地址。在地址变换过程中，第一阶段的分段变换机制总是使用的，而第二阶段的分页机制则是供选用的。如果没有启用分页机制，那么分段机制产生的线性地址空间就直接映射到无力气的物理地址空间上。物理地址空间定义为处理器在其地址总线上能够产生的地址范围。

总结

线性地址：分页机制开启就是逻辑地址和物理地址的中间层；不开启就是物理地址。

物理地址：是机器中的内存容量范围，也可以当做是机器中实际的内存地址。

逻辑地址：是对程序而言的，也是程序员自己看到的地址。一般以Seg:Offset来表示。

虚拟地址：可以当做逻辑地址，也可以当做线性地址。

面试经验讲座笔记

Posted on 2016-01-03 | In 日记

年前参加了学院的面试经验讲座记了些笔记，正好今天晚上没有心情学习就来记录一下吧

分类

根据工作的类型分为3类

  • 互联网企业
  • 银行软开
  • 研究所

互联网企业

互联网企业着重说一下准备工作和面试流程

内推

重要性

找互联网工作内推很重要，因为内推相当于对一次面试机会

内容

内推面试有两个特点

  • 专业性强
  • 深度优先

内推面试一般都是抓住一个知识点死命往深了挖，知道挖不动，面试官会换问题。

因此，在这半年学习中在侧重学习广度的同时要有针对性地深度学习

准备工作

准备

  • 数据结构
  • 基础知识（推荐：《王道程序员求职宝典》、《剑指offer》等面经）

根据岗位重点学习

例如，

  • 网络：TCP/IP
  • 运维：Unix

实战

增强自己项目经验、实际动手能力

简历

  • 排版清晰（推荐：小四，宋体）
  • 突出项目

面试

面试大致四面三类

  1. 基础面
  2. 项目面（2次）
  3. 知识面

基础面

考计算机基础知识、数据结构和算法

基础知识

计算机基础，编程语言知识。

尤其，面向对象语言，问设计模式

数据结构

  • 表/树/图
  • 几种排序算法

项目面

介绍项目内容和关键技术

因此，在面试之前把自己做的项目梳理一遍，把当时用到的一些不太理解的东西深挖一下

知识面

会发散提问，可以不太会，但是要知道

平时订阅一些互联网技术、新闻，开阔视野

银行技术岗位

部门优先级：总行信息技术部>软件开发中心/数字开发中心>省行相关部门

薪水优先级：建行>中行/农行(12+)>工行(10+)

研究所

抓住专场，提前找好目标

简历要条理清晰，写出你的经历和项目

备注

简历要准备两种，互联网和银行/研究所分开

  • 互联网着重项目（四六级都可以不要）
  • 研究所注重个人信息（年龄，学历，家庭情况）和经历（学生时期的资历）
